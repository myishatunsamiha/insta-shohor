Q. How does javascript work?

Ans: JavaScript is an interpreted(gets executed line by line) and dynamically-typed(type of the variable is identified on the run-time) language. The browser(chrome) has a javaScript engine called v8 inside it and the javaScript code runs on that v8 engine. There is a just-in-time(JIT) compiler inside the v8 engine, it improves the performance and makes it faster. The v8 engine takes the js code and parses it in order to check for any error. If no error is found, the code is turned into an abstract syntax tree(ast). Then, bytecode is generated from the ast. After that, optimization and compilation are done on the bytecode. Finally, the optimized code gets executed on the system.



Q. What does javaScript event loop do?

Ans: JavaScript is a single-threaded language which means it has only one call stack. In this call stack, all the functions and operations are pushed in, executed and popped out in first-in last-out manner. Whenever the stack gets any asynchoronus function(any events, setTimeout, etc.), the asynchronous function is immediated sent to the browser api and after a duration the function is sent to the event queue. All the asynchronous functions get stored in the event queue sequentially. When all the functions and operations get popped out and executed from the call stack, the stack becomes empty. Then the functions from the queue(first-in first-out manner) get into the call stack sequentially and in the call stack each of them gets executed. The event loop keeps track of the call stack and the event queue simultaneously. Whenever the event loop gets the stack empty, it pushes the queue functions (if there is any) into the call stack for processing.
